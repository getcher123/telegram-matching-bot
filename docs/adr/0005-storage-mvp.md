# ADR-0005: SQLite для локального MVP

## Контекст
Для MVP нужна минимальная инфраструктура хранения. Приложение запускается локально или на одном сервере.

## Решение
Использовать SQLite через SQLAlchemy async (aiosqlite). Репозитории скрывают SQL за доменными интерфейсами, что позволит мигрировать на PostgreSQL без изменения бизнес-логики.

## Последствия
- Нет внешней БД для локального запуска.
- Миграции идут через SQLAlchemy.
- Разделение repository layer упрощает замену storage.
- PostgreSQL — post-MVP задача.

## Отклонённые альтернативы
- PostgreSQL сразу — избыточно для MVP, усложняет локальный запуск.
