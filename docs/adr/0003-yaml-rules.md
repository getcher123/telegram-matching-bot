# ADR-0003: YAML как источник правил

## Контекст
Правила должны меняться без изменения кода и без перезапуска приложения.

## Решение
Хранить правила, словари и настройки в YAML-файле. ConfigService валидирует через Pydantic, компилирует в runtime-объекты и поддерживает атомарный reload через POST /config/reload.

## Последствия
- Невалидный YAML не ломает активную конфигурацию.
- Каждое решение rule engine привязано к config version (hash).
- Новые правила добавляются правкой YAML, не кода.

## Отклонённые альтернативы
- JSON — менее читаем для словарей и правил.
- TOML — не поддерживает комментарии.
- База данных — усложняет первоначальную настройку.
