# ADR-0004: Детерминированный rule engine без LLM в рантайме

## Контекст
Для MVP критична предсказуемость: одинаковый текст и конфиг должны давать одинаковый результат.

## Решение
Rule engine использует только детерминированную логику: словарные совпадения, числовые сравнения, context window, negative rejection. Никаких LLM-вызовов в pipeline.

## Последствия
- Каждое совпадение объяснимо через evidence.
- Нет зависимости от внешнего API в рантайме.
- Стоимость обработки сообщения минимальна.
- LLM может использоваться только offline для генерации словарей.

## Отклонённые альтернативы
- GPT/LLM в pipeline — недетерминированно, дорого, медленно.
