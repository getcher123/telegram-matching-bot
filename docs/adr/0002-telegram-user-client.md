# ADR-0002: Telegram user-client (Telethon)

## Контекст
Проект требует подключения к Telegram как user-аккаунт (не Bot API), чтобы читать сообщения из групп, куда аккаунт добавлен вручную.

## Решение
Использовать Telethon — зрелый MTProto-клиент с asyncio-поддержкой, авторизацией через код/2FA и событийной моделью.

## Последствия
- Session-файл хранится локально, защищён правами 0600.
- Код авторизации и 2FA-пароль передаются через API, не логируются.
- Для тестов используется mock-клиент.
- Telethon-зависимость изолирована в модуле `app/telegram`.

## Отклонённые альтернативы
- Bot API — не может читать сообщения групп как user.
- pyrogram — менее популярен, хуже документация.
