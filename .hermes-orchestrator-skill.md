## Ключевые принципы
- Детерминированность (без LLM в runtime)
- Объяснимость (evidence + decision trace)
- Security (секреты НЕ в git/логах)
- Atomic reload YAML
- Dedup (один message = один alert)
- Тестируемость (unit на нормализацию/extraction/rules)

## Формат task packet
Включать: task_id, title, priority, role, objective, context, allowed_paths, forbidden_paths, requirements, acceptance_criteria, commands (test/lint/typecheck), expected_outputs, notes_for_agent.

## Проверка результата
1. Есть AGENT_RESULT.md и agent_result.json?
2. Изменения ТОЛЬКО в allowed_paths?
3. Нет секретов в diff?
4. Команды проверки проходят?
5. Acceptance criteria выполнены?
6. Решение: accept / needs_rework / reject
